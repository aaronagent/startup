# Phase 0: Launch Validation Plan

> The design docs say: "不要写一行代码。先做这个。"
> We already built the product. Now we validate demand.

## Strategy

### Step 1: Reddit Curation Post (This Weekend)

**Target:** r/vibecoding (153K+ members)
**Goal:** 50+ upvotes OR 10+ meaningful comments
**Strong signal:** Someone asks "Can I subscribe?" or "Do this weekly"

**Post timing:** Tuesday-Thursday, 9-11am EST
**Cross-post to:** r/sideproject, r/webdev

### Step 2: DM Outreach (Same Day)

After posting, DM 5 tool makers from the list:

> "Hey, I noticed your tool [NAME]. When you built it, what was the
> hardest part about getting people to know about it? If there was a
> tool that auto-generated your landing page, Reddit post, and Twitter
> thread from just your URL, would you use it?"

Record answers in this doc.

### Step 3: Twitter Thread (Next Day)

Thread format:
1. "I curated the 10 best vibe-coded tools I found this week 🧵"
2-9. One tool per tweet with screenshot
10. "Building a platform to discover more: [VibeBoard link]"
11. "Also built LaunchKit — paste your URL, get launch copy for every platform: [LaunchKit link]"

### Step 4: Measure (1 Week Later)

| Metric | Target | Actual |
|--------|--------|--------|
| Reddit upvotes | 50+ | |
| Reddit comments | 10+ | |
| "Can I subscribe?" requests | 1+ | |
| VibeBoard page views | 100+ | |
| LaunchKit usage | 20+ | |
| DM response rate | 3/5 | |
| DM "would use LaunchKit" | 2/5 | |

### Decision Framework

- **Full Pass:** Reddit 50+ votes AND 3+ DMs positive → Build Phase 1
- **Partial Pass:** One side hits target → Iterate 2 more weeks
- **Fail:** Both below target → Pivot or abandon

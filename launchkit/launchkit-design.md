## Problem Statement

Vibe coders 做出了工具，但不知道怎么推广。不是缺渠道（Reddit/Twitter/HN/PH 都在），
而是缺知识和动力。大部分人从没在任何渠道上尝试过分发。

LaunchKit 把"推广"这件事简化到极致：你输入你的工具 URL，它帮你生成 landing page、
社交媒体帖子、提交到各大平台的文案，甚至直接帮你发。

## Demand Evidence

**目前为零。** 与 VibeBoard 相同，纯假设阶段。

唯一的间接证据：independent AI second opinion 在审查 VibeBoard 时指出，
"大部分 vibe coders 的问题不是缺少平台，而是缺少分发的知识和动力"，
建议做"distribution toolkit/guide"而不是平台。LaunchKit 正好是这个方向。

## Status Quo

Vibe coder 做完一个工具后的推广流程（如果他们推的话）：

1. 手动写一个 landing page（或者不写）
2. 手动写 Reddit 帖子
3. 手动写 Twitter 帖子
4. 手动提交到 Product Hunt（流程重，需要 hunter）
5. 手动截图、录屏
6. 每个渠道的格式、字数、风格都不同

大部分人在第 1 步就停了。不是不想做，是太麻烦了。

## Target User

用 AI 做出了一个可用的工具/网页/软件的 vibe coder。他们有一个 URL，
工具是 live 的，但没人知道。他们想推广但不知道从何开始，或者觉得太耗时。

## Premises

1. 大部分 vibe coders 不推广的原因是"太麻烦"而不是"没渠道"。— 需要验证
2. 如果把推广简化到"粘贴 URL → 获得所有推广素材"，会有显著更多的人去做分发。— 需要验证
3. AI 可以从一个 URL 自动提取工具的功能描述、截图、核心卖点，
   并生成适配不同渠道的推广文案。— 技术上可行

## Cross-Model Perspective

来自 VibeBoard session 的 independent Claude subagent cold read：

> "如果调查 100 个 vibe coders，70%+ 从未尝试过分发。障碍是动机和知识，不是结构。
> 正确的产品可能是 distribution toolkit/guide，而不是另一个平台。"

LaunchKit 直接回应了这个洞察。

## Recommended Approach

### 核心功能：URL → Launch Package

用户输入一个 URL，LaunchKit 自动：

1. **抓取工具信息：** 爬取页面，提取标题、描述、功能、截图
2. **生成 Landing Page：** 一个美观的单页介绍页，带 CTA（"Try it" / "Star on GitHub"）
3. **生成社交帖子：** 适配 Reddit（标题 + 正文）、Twitter（280字以内 + 话题标签）、
   Hacker News（Show HN 格式）的推广文案
4. **生成提交文案：** Product Hunt 提交用的 tagline + description
5. **一键分享（Phase 2）：** 直接发布到各个平台（需要 OAuth 授权）

### 执行路线图

**Phase 0: 零代码验证（这个周末）**

找到 5 个 vibe coder（Reddit/Discord），问他们：
"如果有一个工具，你粘贴你的项目 URL，它帮你生成 landing page 和所有社交平台的推广帖子，
你会用吗？" 收集反馈。

**Phase 1: MVP（第 1-3 周）**

技术栈：Next.js + Vercel + Claude API (用于文案生成) + Puppeteer (用于截图)

功能：
1. 输入 URL → 自动抓取页面信息（Puppeteer headless 截图 + 内容提取）
2. 调用 Claude API 生成推广文案（Reddit / Twitter / Product Hunt 格式）
3. 生成一个静态 landing page（可分享的 URL）
4. 所有生成内容可复制、可编辑

不做的事情：
- 不做一键发布（Phase 2）
- 不做用户系统（用完即走）
- 不做付费

验证标准（3 周）：
- **Full Pass:** 100+ 用户使用（输入 URL 并生成了 launch package），30%+ 复制了至少一段文案
- **Partial Pass:** 50+ 用户使用但复制率低 → 文案质量问题，迭代 prompt
- **Fail:** <50 用户 → 需求不成立

**Phase 2: 一键分发 + 付费（第 4-8 周）**

Phase 1 验证通过后：
1. OAuth 集成（Reddit API、Twitter API）
2. 一键发布到 Reddit / Twitter
3. Freemium 模型：免费生成文案，付费($5/launch) 解锁一键发布 + 高级 landing page 模板

**Phase 3: 与 VibeBoard 整合（第 9+ 周）**

如果两个项目都通过验证：
- LaunchKit 生成的 landing page 自动提交到 VibeBoard
- VibeBoard 上的工具可以一键用 LaunchKit 生成推广包
- 形成闭环：发现（VibeBoard）→ 推广（LaunchKit）→ 更多发现

### 差异化

1. **AI 驱动的文案生成** — 不是模板填空，是根据你的工具内容智能生成
2. **多渠道适配** — 一次输入，生成适配 5+ 渠道的内容
3. **Landing page 即服务** — 很多 vibe coders 不会做 landing page，这帮他们解决了
4. **与 VibeBoard 的协同** — 如果 VibeBoard 起来了，LaunchKit 是自然的推广入口

## Open Questions

1. **定价模型：** Free tier 够不够用？付费 $5/launch 还是月订阅？
2. **文案质量：** AI 生成的推广文案对转化率的真实影响？需要 A/B 测试
3. **平台 API 限制：** Reddit/Twitter API 的 rate limits 和 ToS 是否允许自动发帖？
4. **Landing page 托管：** 生成的 landing page 托管在哪？LaunchKit 域名下还是用户自定义域名？

## Success Criteria

Phase 0：
- [ ] 5+ vibe coders 明确表示"会用"并描述了具体场景
- [ ] 无人感兴趣 → 方向不对

Phase 1：
- [ ] 3 周内 100+ 用户使用
- [ ] 文案复制率 > 30%
- [ ] 至少 1 个用户用 LaunchKit 生成的内容实际发布到了某个平台

Phase 2：
- [ ] 月付费用户 50+
- [ ] 一键发布使用率 > 20%

## Distribution Plan

LaunchKit 本身的分发：
- Product Hunt launch（吃自己的狗粮，用 LaunchKit 给自己做推广）
- r/vibecoding, r/sideproject 帖子
- Twitter build-in-public
- 部署 Vercel，域名 launchkit.dev 或类似

## Dependencies

1. Claude API key（文案生成）
2. Puppeteer / headless browser 部署方案（页面截图和内容提取）
3. Vercel 账号
4. 域名注册

## The Assignment

**和 VibeBoard 的 Assignment 可以同时做：**

在你发 Reddit 帖子策展 10 个 vibe coded 工具的时候，顺便私信其中 5 个工具的 maker，
问一个问题：

"Hey, I noticed your tool. When you built it, what was the hardest part about
getting people to know about it? If there was a tool that auto-generated your
landing page, Reddit post, and Twitter thread from just your URL, would you use it?"

把回答记录下来。这就是 LaunchKit 的需求验证。

## What I noticed

- LaunchKit 直接回应了 independent AI 的核心洞察（问题是动力和知识，不是渠道）
- 如果 VibeBoard 验证失败但 LaunchKit 的 DM 验证成功，LaunchKit 可以独立存在
- 如果两个都验证成功，Phase 3 的整合会创造一个比各自更强的飞轮
- 最大风险：AI 生成的推广文案是否真的比人写的好？如果不够好，核心价值主张崩塌
